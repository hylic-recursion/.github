# hylic-recursion

`hylic` is a Rust library for tree-shaped recursive computation — directory traversal, dependency resolution, expression evaluation, configuration inheritance, anywhere the pattern is *process each node, combine with the results from its children*. It structures that pattern into three independent values: a `Fold` saying what to compute at each node, a `Treeish` saying how a node yields its children, and an `Executor` driving the recursion. The same fold runs sequentially or in parallel by swapping the executor; over a struct tree or a flat adjacency list by swapping the treeish; with transformations like `map`, `contramap`, `filter`, and `memoize` without rewriting the closures.

Parallelism comes from `Funnel`, a work-stealing engine bundled in the core crate. Its three policy axes — queue topology, accumulation strategy, wake — all monomorphise at compile time, so there is no runtime dispatch on strategy choice. On the published 14-workload [Matrix benchmark](https://hylic-recursion.github.io/hylic-docs/cookbook/benchmarks.html#matrix), a Funnel variant wins ten rows outright against handrolled Rayon and a scoped pool, and lands within a few percent of the winner on the rest.

For programs that prefer building computations through chained methods rather than composing values by hand, [`hylic-pipeline`](https://github.com/hylic-recursion/hylic-pipeline) is a typestate builder layered on the same primitives — `.wrap_init(...).zipmap(...).run(...)` instead of values composed step by step. Both surfaces share the same fold-treeish-executor decomposition underneath.

The long-form material lives at <https://hylic-recursion.github.io/hylic-docs>: a [quick start](https://hylic-recursion.github.io/hylic-docs/quickstart.html), [the recursive pattern](https://hylic-recursion.github.io/hylic-docs/concepts/separation.html) explained, the [Funnel executor](https://hylic-recursion.github.io/hylic-docs/funnel/overview.html) deep-dive, the [interactive benchmark viewer](https://hylic-recursion.github.io/hylic-docs/cookbook/benchmarks.html), and a [cookbook](https://hylic-recursion.github.io/hylic-docs/cookbook/fibonacci.html) of worked examples (filesystem summary, expression evaluation, module resolution, configuration inheritance, cycle detection, parallel execution). The repositories that ship the code:

- [`hylic`](https://github.com/hylic-recursion/hylic) — core library: fold, treeish, executor, and the bundled Funnel parallel engine. Published on [crates.io](https://crates.io/crates/hylic).
- [`hylic-pipeline`](https://github.com/hylic-recursion/hylic-pipeline) — chainable typestate builder over the core primitives. Published on [crates.io](https://crates.io/crates/hylic-pipeline).
- [`hylic-benchmark`](https://github.com/hylic-recursion/hylic-benchmark) — Criterion harness behind the published numbers.
- [`hylic-docs`](https://github.com/hylic-recursion/hylic-docs) — mdBook source for the documentation site.
